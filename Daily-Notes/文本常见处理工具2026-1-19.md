## vim
#### 题目： 在一个拥有 1000 行代码的文件中，如果你发现第 500 行有一个变量名写错了，你有哪些方法可以快速跳转到该行？请至少说出三种。  
>命令行模式： 输入 :500 然后回车。  
普通模式： 输入 500G 或 500gg。    
搜索定位： 如果记得具体的变量名，直接输入 /变量名。  
#### 题目： 假设你有一份配置文件，里面有 20 行以 http:// 开头的地址，现在需要把它们全部改为 https://。在不使用插件的情况下，你会怎么做？
>使用底行命令替换：:%s/http:\/\//https:\/\//g（或者:%s#http://#https://#g）

| 符号 | 名称 | 作用 |
| :--- | :--- | :--- |
| **`:`** | 命令行模式 | 进入 Vim 的命令输入状态。 |
| **`%`** | 范围限定 | 表示对**全文**（所有行）进行操作。 |
| **`s`** | 替换命令 | `substitute` 的缩写，表示执行替换操作。 |
| **`/`** | 分隔符 | 用来区分“查找内容”和“替换内容”。 |
| **`http:\/\/`** | 查找内容 | 这里的 `\` 是转义字符。因为 `/` 已被用作分隔符，所以原始路径里的 `/` 必须写成 `\/` 才能被正确识别。 |
| **`https:\/\/`** | 替换内容 | 将匹配到的内容替换成安全协议头 `https://`。 |
| **`g`** | 全局标志 | `global` 的缩写。表示如果一行内出现多次匹配，则**全部替换**；若不加 `g`，则每行只替换第一个。 |

#### 题目： 你正在编辑一个 HTML/XML 文件，光标在以下代码行的 p 字母上： <p class="header">Hello |World</p> 要求： 将 Hello World 快速改为 Welcome，但保留 <p...> 标签和双引号内的 class 属性。请给出最简按键。

>光标移入标签 dit （change in tag）

#### 题目： 你正在编辑一个 HTML/XML 文件，光标在以下代码行的 p 字母上： <p class="head|er">Hello World</p> 要求： 将 header 删除。请给出最简按键。

>光标移入双引号" di"
#### 题目： 你正在编辑一个 HTML/XML 文件，光标在以下代码行的 p 字母上： <p class="head|er">Hello World</p> 要求： 将 header 删除，并同时进入插入模式。请给出最简按键。
>光标移入双引号" ci"
#### 题目： 你正在重构一段代码，需要删除光标所在的整个函数体（花括号 {} 之间的所有内容），但是保留花括号本身。

>di{
